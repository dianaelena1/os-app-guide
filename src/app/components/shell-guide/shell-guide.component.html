Ghid rapid pentru studenti

Scopul acestui ghid este să vă ajute să înțelegeți mai ușor cum gândiți comenzile în shell scripting, nu doar să le memorați.

  • 1. Începeți de la problemă.
    Întrebați-vă mai întâi: ce vreau să obțin?
  • 2. Gândiți input-ul și output-ul.
    Ce primește comanda? Un fișier, un text, o listă de linii?
    Ce produce? Linii afișate, un număr, un text modificat?
  • 3. Alegeți tool-ul potrivit.
    grep caută,
    sed modifică text,
    awk lucrează pe coloane,
    chmod schimbă permisiuni.
  • 4. Mergeți de la simplu la combinat.
    Întâi scrieți comanda de bază, apoi adăugați opțiuni, pipe sau redirectare.
  • 5. Testați cu exemple mici.
    Este mai ușor să înțelegeți o comandă pe 3–4 linii simple decât pe un fișier mare.
  • 6. Nu memorați mecanic.
    Încercați să înțelegeți de ce funcționează o comandă, nu doar cum se scrie.

1. Overview

În shell scripting, multe probleme se rezolvă prin combinarea unor unelte simple. Gândește fiecare comandă ca pe o piesă cu un rol clar:

Tool La ce e bună Întrebare utilă pentru student
chmod schimbă permisiunile Cine are voie să citească, scrie sau execute?
grep caută linii după pattern Ce linii vrem să găsim?
sed editează textul în flux Ce vrem să transformăm?
awk lucrează pe câmpuri/coloane Ce coloane vrem să extragem sau calculăm?

Fir logic bun

  1. identific problema
  2. aleg comanda principală
  3. adaug opțiuni
  4. combin cu pipe sau redirectare

Exemplu de gândire

cat log.txt | grep -i error | wc -l

Mai bine, fără cat inutil:

grep -i error log.txt | wc -l

3. chmod – permisiuni

chmod modifică permisiunile unui fișier sau director.

Simbolic

chmod u+x script.sh
chmod g-w file.txt
chmod o+r notes.txt
  • u = user
  • g = group
  • o = others
  • + adaugă, - elimină

Numeric

chmod 755 script.sh
chmod 644 report.txt
  • r=4, w=2, x=1
  • 7 = rwx
  • 6 = rw-
  • 5 = r-x

Exemple explicabile rapid

chmod +x script.sh     # face scriptul executabil
chmod 755 script.sh    # owner: rwx, group: r-x, others: r-x
chmod 644 file.txt     # owner: rw-, group: r--, others: r--

Observatie!

Studenții confundă permisiunea de execuție cu dreptul de citire. Un fișier poate fi citibil fără să fie executabil.

4. grep – căutare în text

grep este o comandă Unix folosită pentru a căuta un pattern de text într-un fișier sau într-un flux de date.

Sintaxă

grep [opțiuni] pattern [fișier]

Pattern-ul poate fi un cuvânt simplu sau o expresie regulată.

Exemple de bază

grep "sequence" file_1.txt
grep sequence file_1.txt file_2.txt
echo "This is a test" | grep "is"

grep poate căuta într-un fișier, în mai multe fișiere sau în output-ul altei comenzi.

Opțiuni foarte importante

-i
ignoră diferența dintre litere mari și mici
-r
caută recursiv în directoare
-l
afișează doar numele fișierelor care au match
-c
afișează numărul de potriviri
-w
potrivește doar cuvântul întreg
-n
afișează și numărul liniei
-v
afișează liniile care NU conțin patternul
-E
folosește expresii regulate extinse
-A n
afișează n linii după match
-B n
afișează n linii înainte de match
-C n
afișează n linii înainte și după match

Exemple utile

grep -i the file_1.txt file_2.txt
grep -c the file_1.txt file_2.txt
grep -rl 1 ~/lab_5
grep -w 1 file_1.txt file_2.txt
grep -n error log.txt
grep -v error log.txt
grep -E "(dog)+" dog_and_cat.txt

Lucrul cu directoare

grep 1 -r ~/lab_5
grep 1 -rl ~/lab_5

Cu -r cauți în directoare. Cu -l vezi doar fișierele care conțin cel puțin un match.

Case sensitive vs. case insensitive

grep the file_1.txt file_2.txt
grep The file_1.txt file_2.txt
grep -i the file_1.txt file_2.txt

În Unix, căutarea este case sensitive. Opțiunea -i ignoră diferența dintre litere mari și mici.

Regex esențiale în grep

Pattern Semnificație Exemplu
. orice caracter grep "test."
\. punct literal grep "test\."
[abcd] un caracter din mulțime grep "[abcd]" file.txt
[^abcd] orice caracter, mai puțin cele din mulțime grep "[^abcd]" file.txt
[0-9] orice cifră între 0 și 9 grep "[0-9]" employees.txt
^text linia începe cu text grep "^Victor" employees.txt
text$ linia se termină cu text grep "9$" employees.txt
\b limită de cuvânt grep -i "\blo" file.txt

Operatori de repetare

grep "dog\?" dog_and_cat.txt
grep "dog\+" dog_and_cat.txt
grep "dog\{2\}" dog_and_cat.txt
grep -E "(dog)+" dog_and_cat.txt
grep -E "(dog){2,3}" dog_and_cat.txt

Basic regex

Fără -E, unele caractere speciale trebuie escapate: 

\( \)
\+
\?
\{m,n\}

Extended regex

Cu -E, sintaxa e mai simplă:

( )
+
?
{m,n}

Context în jurul unui match

grep -E -B 2 "snoop" dog_and_cat_lines.txt
grep -E -A 2 "snoop" dog_and_cat_lines.txt
grep -E -C 2 "snoop" dog_and_cat_lines.txt

Context în jurul unui match

grep -E -B 2 "snoop" dog_and_cat_lines.txt
grep -E -A 2 "snoop" dog_and_cat_lines.txt
grep -E -C 2 "snoop" dog_and_cat_lines.txt

Greșeli frecvente

Studenții confundă des căutarea într-un fișier cu căutarea într-un director și uită când trebuie folosit -r. De asemenea, -i și -w sunt foarte utile, dar deseori uitate.

5. sed – editare de text în flux

sed vine de la stream editor și este folosit pentru a transforma, filtra sau edita text direct din linia de comandă.

Sintaxă

sed [-n] [-e] 'command' [input-file]
  • -n = suprimă afișarea automată
  • -e = permite mai multe expresii
  • -f = citește scriptul dintr-un fișier
  • -i = modifică fișierul direct

Idee importantă

Fără -n, sed afișează automat liniile procesate. De aceea, uneori pare că o linie este afișată de două ori. 

sed 'p' employees.txt
sed -n 'p' employees.txt

Selectarea liniilor

sed -n '2p' employees.txt
sed -n '2,5p' employees.txt
sed '$p' employees.txt
sed -n '/Tudor/p' employees.txt
sed -n '/Ion/,/Victor/p' employees.txt
sed -n -e '2p' -e '5p' employees.txt

Exemple cu linii și intervale

Comandă Semnificație
sed -n '2p' file.txt afișează linia 2
sed -n '2,5p' file.txt afișează liniile 2–5
sed '$p' file.txt afișează ultima linie
sed -n '/Tudor/p' file.txt afișează liniile care conțin Tudor
sed -n '/Ion/,/Victor/p' file.txt afișează intervalul dintre două pattern-uri

Ștergerea liniilor

sed 'd' employees.txt
sed '4d' employees.txt
sed '/Tudor/d' employees.txt
sed '2,10d' employees.txt
sed '/Tudor/,$d' employees.txt
sed -e '2d' -e '5d' employees.txt

Substituția

sed 's/Tudor/Todorutz/' employees.txt
sed 's/19/18/g' employees.txt
sed -n 's/1931/1932/p' employees.txt
sed -n 's/1931/1932/gp' employees.txt

s/vechi/nou/ înlocuiește prima apariție de pe fiecare linie. Dacă adaugi g, înlocuiești toate aparițiile de pe linia respectivă.

Substituție pe intervale

sed -e "2,6p" -e "2,6s/dog/cat/" sed_dog_and_cat.txt
sed -e "2,6p" -e "2,6s/dog/cat/g" -e "2,6s/hot/cold/" sed_dog_and_cat.txt

Grupuri și backreferences

sed -n 's/\(Ion\)el/\1ut/p' employees.txt
sed -n 's/\(Vasi\)le\b/\1lica/p' employees.txt
sed -E -e 'p' -e 's/([0-9])\1$/\1\1.5/' employees.txt

Cu paranteze poți grupa o secvență și o poți refolosi mai târziu prin \1, \2 etc.

Alte comenzi utile

sed '3a line added' employees.txt
sed '2c retired employee' employees.txt
sed '1i \t\t\t PERSONAL DATA' employees.txt
sed '5q' employees.txt
sed '3r sed_dog_and_cat.txt' employees.txt
sed -n 'w employee.bak' employees.txt
sed -e '5q' -e 'w employee_1.txt' employees.txt
Comandă Rol
a adaugă text după linie
i inserează text înainte de linie
c înlocuiește linia completă
q oprește procesarea
r citește și inserează conținut din alt fișier
w scrie rezultatul într-un fișier
p afișează linia
d șterge linia
s substituie text

Atenție la -i

Opțiunea -i modifică fișierul direct. Este foarte utilă, dar poate fi periculoasă la început, pentru că schimbă conținutul fără să mai fie nevoie de redirectare sau alt fișier.

6. awk – coloane și procesare

awk lucrează foarte bine când datele au structură pe câmpuri. 

awk '{print $1}' data.txt
awk '{print $1, $3}' data.txt
awk '$3 > 100 {print $1, $3}' sales.txt
awk -F: '{print $1}' /etc/passwd
awk '{sum += $2} END {print sum}' prices.txt

Ideea de bază

  • $1 = primul câmp
  • $2 = al doilea câmp
  • NF = numărul de câmpuri
  • NR = numărul liniei

Cu separator personalizat

awk -F, '{print $1, $4}' file.csv
awk -F: '{print $1}' /etc/passwd

Când aleg awk în loc de grep sau sed?

Când te interesează o anumită coloană, un calcul sau o condiție pe câmpuri.

2. Basics

Arhitectura Unix

Într-un sistem Unix/Linux, utilizatorul interacționează cu shell-ul, care rulează în user space. Shell-ul transmite comenzile către kernel, iar kernel-ul comunică direct cu hardware-ul.

Structura unei comenzi

comanda [opțiuni] [valori]
  • comanda – ex: ls, cd
  • opțiuni – ex: -l, -a
  • valori – ex: fișiere sau directoare
ls -l /etc
mkdir test
touch file.txt

Comenzi de bază

pwd 
pwd

Directorul curent

cd 
cd ..
cd ~
cd /

Navigare

ls 
ls
ls -l
ls -a

Listare fișiere

man 
man ls

Manual comenzi

Data Streams

Un program primește date prin stdin, afișează rezultatul normal prin stdout și afișează erorile prin stderr.

Stream Descriere
stdin (0) Input – tastatura
stdout (1) Output – ecran
stderr (2) Erori – ecran

Redirection

date > file.txt
date >> file.txt
sort < file1 > file2
  • > – suprascrie fișier
  • >> – adaugă la final
  • < – input din fișier

Pipes – legătura între comenzi

Un pipe este un mecanism de comunicare între procese (IPC). În shell, este reprezentat prin simbolul |.

  • Output-ul unei comenzi (stdout) devine input pentru alta (stdin)
  • Permite combinarea comenzilor în pipeline-uri
  • Evită folosirea fișierelor intermediare

Exemplu simplu

grep error log.txt | wc -l

Caută liniile cu "error" și le numără

Exemplu din slide

cat file1 file2 file3 | sort > dev/lp

Concatenează fișierele, sortează rezultatul și îl trimite mai departe

Greșeală frecventă: studenții folosesc fișiere temporare în loc de pipe-uri. De multe ori, un | rezolvă problema mai elegant.

De ce folosim pipe?

cat file.txt | grep error | wc -l

Combinăm comenzi simple pentru a rezolva probleme complexe.

Construcții Bash

Shebang 
#!/bin/bash
Variabile 
name="Diana"
echo $name
Input 
read name
echo "Salut, $name"
If 
if [ -f file.txt ]; then
  echo "Exista"
fi
For 
for x in a b c; do
  echo $x
done

Diferențe scurte care ajută

Concept Rol
| pipe între comenzi

>> append în fișier
$() execută comandă
${} variabile

3. ls – listarea fișierelor și directoarelor

ls este comanda folosită pentru a afișa conținutul unui director. Este una dintre cele mai importante comenzi de bază în Linux.

Sistemul de fișiere Linux este organizat ca un arbore, pornind de la / (root). Fiecare director poate conține alte directoare sau fișiere.

Sintaxă de bază

ls
  ls folder/
  ls file1 file2

Poți lista directorul curent, un alt director sau mai multe fișiere și directoare.

Opțiuni utile

ls -l
  ls -a
  ls -lh
  ls -la
  ls -R
  • -l = format lung
  • -a = afișează și fișierele ascunse
  • -h = dimensiuni ușor de citit
  • -R = listare recursivă

Exemple importante

ls                # afișează conținutul directorului curent
  ls -l             # afișează detalii despre fișiere
  ls -a             # afișează și fișierele ascunse
  ls -la            # combinație foarte folosită
  ls -lh            # dimensiuni în format ușor de citit
  ls /home/student  # listează conținutul altui director

Cum citești rezultatul lui ls -l

Exemplu Semnificație
-rw-r--r-- permisiunile fișierului
1 numărul de linkuri
diana owner
studenti group
245 dimensiunea fișierului
Apr 18 data ultimei modificări
script.sh numele fișierului

Legătura cu chmod

Comanda ls -l este foarte utilă împreună cu chmod, pentru că îți arată permisiunile înainte și după ce le modifici.

Greșeală frecventă

Mulți studenți folosesc ls, dar nu citesc și nu interpretează coloanele din ls -l. Tocmai acolo se vede informația importantă despre permisiuni.
